# e世代生命密碼分析儀

以「生日生命靈數」為主功能的繁體中文數理分析工具，另提供生日九宮、數字磁場、台灣身分證流年、本機歷史、三數取卦及獨立的邵康節易學專頁。所有結果都顯示規則版本與可核對算式。

## 公開網站

https://endfew-ai.github.io/e-shidai-life-code/

邵康節易學專頁：

https://endfew-ai.github.io/e-shidai-life-code/kangjie.html

專頁入口密碼：`0000`。同一個瀏覽器分頁通過後會保留到分頁工作階段結束。這是 GitHub Pages 前端入口鎖，適合避免一般誤入，不是伺服器級機密保護。

首頁的「三數取卦」也使用密碼 `0000`；錯誤或取消時會留在原本功能，通過後才顯示三數輸入區。這同樣是瀏覽器端簡易入口限制，不是機密資料防護。

首頁右上角顯示「累積造訪」次數。同一分頁工作階段最多增加一次，重新整理只讀取目前數字；這是約略的頁面開啟次數，不是去重後的實際使用人數，也不作為營運或安全依據。

同一個網址會依螢幕寬度自動適配手機與電腦，不需要切換版本。

## 視覺方向

- 採 B「玄星觀象」：深靛黑、古金、精密星軌與宋明書卷感。
- 主要視覺與分析模組使用專案專屬生成圖，不把易經符號當成首頁主題。
- 品牌、主題與所有固定結構標題各自使用逐字校對的黃金墨水毛筆字模；Hero 星軌背景與主標語也是兩個獨立檔案。HTML 同時保留隱藏真實文字，搜尋與讀屏不受影響。
- 生日命碼是第一層入口，數字頻譜為次要功能，三數取卦為補充工具，邵康節易學則使用同網域獨立專頁。
- 新專頁的入口、主題、主標、分頁、方法、表單、卦象角色與原文區標題均為各自獨立生成、逐字校對、透明去背的黃金墨水毛筆字模。
- 動態算出的卦名、數值、年份與公式保留為可選取的真實文字，避免 AI 假字及資料失真。
- 桌機與手機共用同一個高密度工作台；主頁四個入口在手機採 2×2 排列，完整長文採原生可展開區塊。
- 參考圖反推母提示詞與各模組 Image2 提示詞收錄在 [`AI_MODULE_PROMPTS.md`](AI_MODULE_PROMPTS.md)。

## 隱私與使用方式

- 生日、身分證、密碼與分析數字只在使用者自己的瀏覽器內計算，不會送到分析伺服器。
- 頁面開啟時會向 CounterAPI V1 送出一次不含上述輸入內容的造訪請求；外部服務仍會接收一般網路請求附帶的連線資料。服務失效時只把計數顯示為不可用，不影響分析功能。V1 是免登入的公開計數器，數字可能被外部更動，長期可用性亦不保證，詳見 [CounterAPI V1 文件](https://docs.counterapi.dev/api/endpoints/v1/)。
- 生日模式只需要西元出生日期，不需要姓名或時辰。
- 身分證入口會先用官方字母代碼與檢查碼做邏輯檢查，再以獨立的 A=01～Z=26 民俗規則轉換。兩套字母表不共用；檢查通過也不代表號碼已配發或能確認持有人。
- 身分證輸入框預設為密碼欄位；完整字號不寫入網址、console 或本機歷史。歷史最多保留 20 筆遮罩摘要，完整轉換序列、配對與時間軸不保存；結果中的相鄰數字也預設遮罩，只有使用者主動點選時才短暫顯示。
- 個人色彩指引依 Cheiro《Cheiro's Book of Numbers》的出生日數 1～9 色群顯示傳統主色；生命路徑色與態度數色均明列為本站延伸，HEX 是螢幕用近似轉譯。
- 自訂序列可分手機、車牌／門牌及一般英數序列；英文字母依 A=01～Z=26 保留前導零，並逐一記錄來源字元。
- 三數取卦只做本卦、互卦、動爻與變卦的固定計算；附 64 卦古籍本文，不提供 AI 解卦或吉凶判斷。
- 邵康節專頁分開處理年月日時、物數、雙段聲數與十一字以上字數法，不把生日或任意號碼冒充通用古法。年月日時會讀取裝置當下時間與時區，自動換算年支、農曆月日及時支，並保留手動修正。
- 皇極經世只做元、會、運、世、年的時間長度分解，不輸出有爭議的西元定位或現代事件預言。
- 內容屬文化娛樂與自我反思用途，不是科學個人色彩測驗、醫療、心理或其他專業判斷。

## 採用規則

- 新版預設 `uploaded-material-v2`：將 `YYYYMMDD` 全部八個數字直接相加，主數預設不保留，最後化簡至 1～9。
- 舊版 `legacy-project-v1`：保留原站月、日、年分段化簡與 11／22／33 主數，可在規則設定切回，不會刪除。
- 主數模式：可選不保留、保留 11／22／33，或明確指定自訂主數；人格摘要一律依化簡後 1～9 基底。
- 九宮分布：新版預設使用生日原始數字、排除 0，依 `1-2-3／4-5-6／7-8-9` 顯示，並評估 8 條主線、4 條副線與 1 條情緒壓力組合。亦可選擇加入生命靈數基底或切回舊版洛書版位。
- 八大磁場：使用完整 64 組有向配對，從左到右建立所有相鄰滑動視窗；0／5 可選只照錄、橋接修飾或舊版未定義。
- 身分證命格數列：字母先依 A=01～Z=26 轉換。若字母碼為 01～09，命格分析只移除最前方的補位 0；J=10 至 Z=26、身分證後九碼及其他位置的 0 都完整保留。命格數列採相鄰磁場分析，不加總成單一數字。
- 身分證時間軸：可明確選擇首段 10 年、首段 13 年、5 年循環、教材原表照錄或舊版無公式。教材區間與配對數量不一致時會產生警告，不會靜默截斷。
- 身分證人生階段永遠使用完整字母兩位碼與後九碼，和命格數列的補位 0 規則分開處理；主要出現磁場只是統計摘要，不冒充單一命格數。
- 人生階段細解：每段可展開查看階段主題、可觀察、可運用、需要留意、前段轉接與分類依據；0／5 維持未分類，橋接只作補充，沒有教材規則時不自行預測。
- 三數取卦：第一數取上卦、第二數取下卦、第三數取動爻；使用固定八卦與六十四卦表。
- 年月日時：年支加農曆月、日取上卦，再加時支取下卦及動爻；預設依裝置當下時間自動帶入，閏月、子初換日與年界仍由使用者依採用曆法確認。
- 物數、聲數、字數：依《梅花易數》現行本各自獨立的入口與算例實作。
- 皇極尺度：一世 30 年、一運 360 年、一會 10,800 年、一元 129,600 年，只作時間單位換算。

## 本機開啟

因靜態版使用 ES Module，請透過本機 HTTP 伺服器預覽，不要直接用 `file://` 開啟 `index.html`：

```powershell
npx --yes serve .
```

開發版本可使用：

```powershell
npm ci
npm run dev
```

完整驗證：

```powershell
npm test
npm run lint
npm run typecheck:contracts
npx playwright test -c playwright.config.mjs
python scripts\audit-brush-assets.py --contact-sheet output\brush-title-contact-sheet.jpg
```

演算法稽核、規格、來源、遷移與測試矩陣位於 [`docs/`](docs/)。
