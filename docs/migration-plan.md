# 生命靈數新版遷移計畫

日期：2026-07-23

## 一、遷移結論

本次屬「演算法與介面相容遷移」，不是資料庫搬遷。

盤點結果：

- `db/schema.ts` 為空；
- `.openai/hosting.json` 的 D1、R2 未啟用；
- 舊站沒有分析歷史資料表；
- 舊站沒有規則設定資料表；
- 未發現需要搬移的本機歷史 key。

因此：

```text
既有使用者分析資料遷移：0 筆
既有使用者規則設定遷移：0 筆
```

不得用測試資料或新產生資料冒充遷移成果。

## 二、遷移目標

1. 保留既有生日分析、色彩、易經與邵康節入口。
2. 將生命靈數計算預設改為可追溯的完整生日八位數規則。
3. 保留舊版分段算法供重算與比較。
4. 新增九宮連線、磁場、身分證與時間軸，不偽裝成舊站既有功能。
5. 新增本機設定與遮罩歷史，不上傳敏感資料。
6. 每筆新結果記錄規則版本，避免日後無法重現。

## 三、新舊差異

| 項目 | 舊版 | 新版預設 | 相容方式 |
| --- | --- | --- | --- |
| 生命靈數 | 月、日、年分段化簡 | 完整生日八位數相加 | 可選 `legacy-project-v1` |
| 主數 | 固定保留 11、22、33 | 預設不保留 | 可切換主數模式 |
| 九宮格 | 原始數字、洛書顯示 | 原始數字、1～9 結構 | `legacy_project` 保留洛書順序 |
| 連線 | 無完整判定 | 主線、次線、強度 | 新增功能，無舊資料可比 |
| 八大磁場 | 無 | 完整 64 組 | legacy 顯示無舊公式 |
| 0／5 | 無公式 | 橋接修飾 | 可選 literal 或 legacy |
| 身分證 | 無 | 官方驗證＋民俗分析 | 新增功能 |
| 人生階段 | 無 | 可選 profile | legacy 不產生時間軸 |
| 命格數 | 無可靠公式 | unresolved | 不猜測 |
| 歷史 | 無 | 本機最多 20 筆遮罩摘要 | 無資料遷移 |

## 四、執行順序

### 階段 1：建立新核心

- 建立版本化規則資料。
- 拆出生日、九宮格、磁場、身分證時間軸、解讀模組。
- 讓每個模組使用純函式與明確輸入。
- 保留 `calculator-core.js` 作舊畫面相容層。

### 階段 2：建立相容規則

- 固定 `legacy-project-v1` 的已知行為。
- 建立新舊差異案例。
- 舊版沒有的功能明確標示 unavailable／unresolved。
- 不以新版結果反填成舊版資料。

### 階段 3：接入 UI

- 首頁提供生日、身分證、手機、自訂序列等獨立入口。
- 結果頁顯示 `ruleSetId`、計算步驟、來源、警告與免責。
- 設定頁明確顯示主數、九宮格、0／5、時間軸 profile。
- 身分證欄位預設遮蔽，顯示結果使用遮罩。

### 階段 4：建立本機設定與歷史

- 設定 key：`e-shidai-numerology-settings-v1`。
- 歷史 key：`e-shidai-numerology-history-v1`。
- 歷史上限 20 筆。
- 儲存前驗證沒有 `normalizedInput` 與 `structuredResult`。
- 提供清除歷史。

### 階段 5：驗證與發布

- 先執行單元測試、型別檢查、lint 與建置。
- 從正式 build 輸出做瀏覽器流程與手機版驗收。
- 確認完整身分證未出現在 localStorage、畫面非必要區域或主控台。
- 驗證 legacy 與新版差異。
- 確認後才發布。

## 五、既有結果處理

因舊站未保存分析結果，沒有可批次更新的紀錄。使用者若持有舊版截圖或抄錄結果，處理方式為：

1. 在設定選擇 `legacy-project-v1` 重算。
2. 記錄舊版結果與新版結果。
3. 以計算步驟說明差異，不能只比較最後一位數。
4. 不將舊結果覆寫為新版，也不宣稱舊結果錯誤；應標示採用不同規則。

## 六、隱私遷移

新版不遷移、匯入或保存完整身分證。歷史紀錄只留：

- 遮罩輸入；
- 輸入類型；
- 規則版本；
- 非敏感摘要；
- 建立時間。

禁止寫入：

- 完整身分證；
- 完整英數轉換序列；
- 可還原身分證的全部配對；
- 完整 `AnalysisResult`；
- 主控台除錯輸出。

若日後導入雲端帳號或資料庫，必須另做隱私影響評估、加密、最小化欄位、刪除機制與使用者同意；本次不能視為已涵蓋。

## 七、復原方式

### 規則復原

- 使用者可將規則切換回 `legacy-project-v1`。
- 設定損壞時回到 `uploaded-material-v2` 安全預設。

### 本機資料復原

- 清除 `e-shidai-numerology-settings-v1`：只會重設規則偏好。
- 清除 `e-shidai-numerology-history-v1`：會刪除本機遮罩歷史，無雲端備份。

清除歷史屬使用者明確操作；程式不應在版本升級時偷偷清除。

### 程式版本復原

- 保留發布前 Git commit／tag。
- 發布異常時回復上一個已驗證版本。
- 不刪除舊規則常數，除非已有替代遷移與回歸證據。

## 八、完成判定

只有同時符合下列條件才可宣稱遷移完成：

- 五個核心模組可獨立測試；
- 新版與 legacy 固定案例均通過；
- 64 組磁場無缺漏或重複；
- 身分證官方驗證與 A=01 分離；
- 時間軸差異會警告；
- 命格數保持 unresolved；
- localStorage 不含完整身分證；
- 桌機與手機主要流程通過；
- 正式建置可啟動；
- 發布版本與驗證 commit 一致。
