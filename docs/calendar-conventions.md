# 農曆與時間慣例

## 自動模式

瀏覽器使用 `Intl.DateTimeFormat("zh-TW-u-ca-chinese")` 取得農曆年、月、日，時區使用 IANA 名稱。預設為 `Asia/Taipei`。若瀏覽器不支援 Chinese Calendar，程式停止自動換算並要求人工輸入。

中央氣象署 A-A0087-001 只用作固定邊界測試與節氣核對，不在 runtime 呼叫外部 API。

官方資料能支持國農曆日期、閏月與節氣核對，不能證明「立春換年」或任一子時換日規則是術數唯一正法；這些差異由 profile 明確記錄。

## Calendar profiles

### `taipei-lunar-new-year-v1`

- 時區：`Asia/Taipei`
- 年界：農曆正月初一
- 閏月：沿用同名月份數
- 換日：民用午夜

### `taipei-lichun-v1`

- 時區：`Asia/Taipei`
- 年界：立春交節時刻
- 預載的中央氣象署固定立春時刻目前只涵蓋已驗證年份。
- 沒有固定資料的年份必須提供 `lichunInstantIso`，否則停止並要求人工確認。

### `taipei-late-zi-next-day-v1`

- 年界：農曆正月初一
- 23 時仍屬子時。
- 農曆日期在 23 時改用次日。

## 閏月

預設閏月沿用同名月份，例如閏二月仍以 2 入數。自訂 profile 可選 next-month-number，但結果必須保留設定，不能靜默變更。

## 手動模式

人工輸入優先於自動轉換。結果會標示：

- `mode = manual`
- 所選時區
- 年界
- 閏月規則
- 子時換日規則
- 人工輸入警告

## 固定測試點

- 2023-03-22：閏二月初一。
- 2025-12-20：農曆冬月初一。
- 2026-01-19：農曆臘月初一。
- 2026-02-17：農曆正月初一。
- 2026-02-04 04:02（臺北）：立春年界測試。
- 23 時：民用午夜與晚子時換日的差異測試。

曆法結果仍須依使用者採用的師承或約定核對，系統不宣稱所有術數流派共用同一慣例。
